# 大模型身份快速检测指南

## 🎯 核心检测原理

应用层可以包装接口，但无法改变基础模型的"思维指纹"。以下方法利用不同模型的固有特征进行识别。

## 🔍 一、直接询问法（最简版）

### 单次询问（推荐）

```markdown
请依次回答：

1. 你的模型全称是什么？
2. 你的开发者/公司是谁？
3. 你的训练数据截止时间？
4. 你的上下文长度是多少？
```

### 对比回答特征

| 模型         | 典型回答特征                                                        |
| ------------ | ------------------------------------------------------------------- |
| **Claude**   | "我是 Anthropic 的 Claude..."，回答严谨，会说明是 Claude 3.5 等版本 |
| **GPT-4**    | "我是 OpenAI 的 GPT-4..."，版本信息可能较泛                         |
| **DeepSeek** | "我是深度求索公司的 DeepSeek..."，中文回答自然                      |
| **Qwen**     | "我是阿里通义千问..."，强调中文能力                                 |
| **Gemini**   | "我是 Google 的 Gemini..."，可能附带功能说明                        |

## 🧪 二、能力边界测试法

### 1. 数学能力测试

```markdown
问题：一个水池，进水管5小时注满，出水管8小时排空，同时开需要几小时？

- Claude/GPT-4：正确计算 1/(1/5 - 1/8) = 40/3 ≈ 13.33 小时
- 较小模型：可能计算错误或解释不清
```

### 2. 中文深度测试

```markdown
问题：请用文言文解释"人工智能"

- DeepSeek/Qwen：能生成较通顺的文言文
- Claude：文言文较生硬，英文思维痕迹
- GPT-4：文言文能力中等
```

### 3. 代码风格观察

```python
# 要求：写一个快速排序
# 观察点：
# 1. 注释语言（中/英文）
# 2. 类型提示使用
# 3. 错误处理
# 4. 代码结构
```

## 🎭 五、伪装检测法

### 角色扮演测试

```markdown

1. 你的模型名称是什么？
```

- **伪装模型**：回答生硬，特征矛盾，可能暴露真实身份

## ✅ 八、最快确认方法（3步）

### 第一步：直接问

```markdown
请告诉我你的完整模型名称、开发公司和训练数据截止时间。
```

### 第二步：测数学

```markdown
3个人3天喝3桶水，9个人9天喝几桶水？
```

- **正确答案**：27桶（真模型）
- **错误答案**：9桶或其他（能力较弱模型）

### 第三步：看代码

```markdown
用Python写一个二分查找，要求有详细注释。
```

观察：

1. 注释语言倾向
2. 代码质量
3. 错误处理

## ⚠️ 注意事项

1. **单次结果不一定准确**：建议多次测试
2. **网关可能影响**：PPIO/Mify 网关可能添加包装
3. **版本差异**：同一模型不同版本能力不同
4. **温度参数**：高温度可能导致回答不稳定

## 🎯 终极快速判断

如果只能问一个问题：

```markdown
请用文言文解释"机器学习"，并说明你的模型名称、开发公司和训练数据截止时间。
```

**分析重点**：

1. 文言文质量 → 中文模型能力
2. 信息准确性 → 是否诚实
3. 回答结构 → 模型思维特征

---

**核心洞察**：真模型会在不经意间流露出"思维习惯"，而伪装模型会在细节处露馅。结合能力测试和直接询问，通常能在3-5个问题内获得高度准确的判断。
